# Learning

Personal notes and practice for frontend interviews — JavaScript, React, browser, CSS, performance, security, and SSR.

## Repository structure

```
Learning/
├── JavaScript/     Tier 1 — JS fundamentals & coding patterns
├── React/          Tier 2–3 — React & Redux
├── Browser/        Tier 4 — DOM, events, storage, CORS
├── CSS/            Tier 5 — layout, box model, specificity
├── Performance/    Tier 6 — optimization, bundling, Lighthouse
├── Security/       Tier 7 — XSS, SQL injection, CSP
└── SSR/            Tier 8 — SSR vs CSR, hydration, Next.js
```

## How to use

Each folder follows the same pattern:

| File | Purpose |
|------|---------|
| `THEORY.md` | **Start here** — detailed Q&A for every interview question |
| `01`–`0N` files | Topic cheat sheets & quick reference |
| `interview-flashcards.js` | One-line review |

## Tier overview

| Tier | Folder | Topics |
|------|--------|--------|
| 1 | `JavaScript/` | Scope, closures, event loop, polyfills |
| 2–3 | `React/` | Hooks, Fiber, Redux, rendering |
| 4 | `Browser/` | URL → page load, DOM/BOM, events, CORS |
| 5 | `CSS/` | Box model, flex/grid, position, CLS |
| 6 | `Performance/` | React perf, code splitting, virtualization |
| 7 | `Security/` | XSS, SQL injection, CSP |
| 8 | `SSR/` | SSR/CSR, hydration, Next.js data fetching |
